1. INTRODUCTION                                             *plz-introduction*

plz.nvim is a unified PR review workflow for Neovim. It replaces gh-dash +
octo.nvim with a single plugin that provides:

- A gh-dash style dashboard for PR triage
- Difftastic-powered structural diffs (syntax-aware, token-level)
- File-centric review with inline threads
- Azure DevOps work item integration

Zero plugin dependencies. Dashboard â†’ diff â†’ review â†’ submit, all without
leaving Neovim.

==============================================================================
2. REQUIREMENTS                                             *plz-requirements*

Required: ~
- Neovim >= 0.10
- `gh` CLI, authenticated (`gh auth login`)
- `difft` (difftastic) for structural diffs

Optional: ~
- `wt` (worktrunk) for worktree-based file exploration with LSP
- `ADO_PAT` environment variable for Azure DevOps work item integration

4. CONFIGURATION                                           *plz-configuration*

Call `setup()` with an optional table. All values shown are defaults: >lua

    require('plz').setup({
      diff = {
        layout = "side-by-side",  -- diff layout
        context = 3,              -- context lines around changes
      },
      worktree = {
        auto_create = true,       -- create worktree on review open
        auto_cleanup = true,      -- remove worktree on review close
      },
      -- Azure DevOps integration (optional)
      ado = {
        org = "",                 -- ADO organization
        project = "",             -- ADO project
        pat_env = "ADO_PAT",     -- env var containing the PAT
      },
    })
<
The plugin works without calling `setup()`. Defaults are applied
automatically.

6. DASHBOARD                                                   *plz-dashboard*

The dashboard (`:Plz`) shows your PRs in a tabbed table view with a
preview pane, similar to gh-dash.

Sections: ~
  Tab 1 - Review Requested
  Tab 2 - My PRs
  Tab 3 - All Open

Dashboard keybindings: ~

    Key             Action ~
    `j` / `k`         Navigate rows
    `<CR>`            Open PR for review
    `o`               Open PR in browser
    `r`               Refresh
    `1` - `3`           Switch tab
    `<Tab>`           Next tab
    `<S-Tab>`         Previous tab
    `/`               Edit filter
    `q`               Close dashboard
    `?`               Toggle help popup

==============================================================================
7. REVIEW                                                       *plz-review*

Pressing `<CR>` on a PR in the dashboard opens the review view with a
file list, summary pane, and side-by-side diff.

The summary pane cycles through three views with `<Tab>`:
  Info â†’ Commits â†’ Description

Review keybindings: ~

    Key             Action ~
    `j` / `k`         Navigate files
    `<CR>`            Open diff / select commit
    `<Tab>`           Cycle summary view
    `]f` / `[f`       Next / previous file (in diff)
    `]h` / `[h`       Next / previous hunk (in diff)
    `o`               Open PR files in browser
    `<BS>` / `q`      Back (commit â†’ PR, diff â†’ files, files â†’ close)
    `?`               Toggle help popup

Commit mode: ~
When viewing the Commits summary, press `<CR>` on a commit to enter
commit mode. This shows only files changed in that commit with diffs
scoped to that single commit. Press `<BS>` or `q` to return to the
full PR view.

==============================================================================
8. DIFF                                                           *plz-diff*

Diffs are powered by difftastic, providing syntax-aware structural diffs
with token-level highlighting. Changed tokens are highlighted individually
rather than entire lines.

The diff view shows two side-by-side buffers (base and head) with
scroll sync enabled.

Navigation: ~
    `]h` / `[h`       Next / previous hunk
    `]f` / `[f`       Next / previous file
